# 2026年 引っ越し先 物件候補の調査

https://psephopaiktes.github.io/move2026/

こちらは人間が書いた初期要求書です。

## 目的
SUUMO掲載サイトについて、以下の物件を引っ越し先候補にしています。以下を調査し、みやすい比較一覧としてまとめ、Webページとして実装してください。

---

## 表示仕様

### 対象物件
- suumo-list.mdにURLがリストアップされています。

### 項目
- 物件名(SUUMOへリンク)
- 掲載写真サムネイル
- 最寄り駅と徒歩時間
- 基本概要
    - 間取り種別・専有面積
    - 築年数
    - 建築構造
    - 階数(`4/5`のように`当該階数/総階数`で表示)
    - 窓の向き
    - 総戸数
- 料金
    - 月額合計
        - 家賃
        - 管理共益費
        - 駐車場
    - 初期費用合計
        - 敷金・礼金・保証金
- 設備
    - 駐車場までの距離
    - 駐輪場の有無
    - ネット環境
    - 敷地内ゴミ置き場の有無
- 近隣施設・周辺情報
    - 最寄りの高速インターチェンジと車でかかる時間
    - SUUMO掲載の`周辺情報`リスト
    - ハザードマップにおける危険性
- その他
    - 契約形態
    - 角部屋かどうか

### デザイン
- テーブルとカードの中間のようなリストビュー
- モダンでクリーン、見やすいデザイン

### 補足
- `近隣施設・周辺情報`など一部の項目についてはSUUMOに掲載されていないので、要外部調査です。Google Mapやハザードマップなどを調べてほしいです。
- 月額・初期費用は合計値のみ表示し、Hover(もしくはクリック)で詳細を見れるようにしてください

## 開発

### 環境
- Viteでセットアップしてください。
- npm start したらブラウザが立ち上がって表示できるようにしてください。
- github pages で公開し、スマホでも見れるようにする予定です。私しか見ないのでno indexにしてください。

### 仕様ドキュメントの保存
- 今後あたらしい項目を追加したい場合にもLLMがすぐ追加できるようにログや仕様などのドキュメントを作成して保存しておくこと
- プロジェクト内の `docs/ai` フォルダの下に適宜ドキュメントを作成し更新・管理する
- 人間が読む想定は少ないので、LLMが理解しやすくなっていればOK

### 作成HTMLのTable仕様
- 定量的な項目で並び替えができること
- 掲載写真はSUUMOから取得し、クリックしたらスライドショー表示できるようにしたい 
- ユーザー評価
    - 5段階で評価できるようにしてください(ローカル環境だけでもOK)
    - 簡単なテキストメモも書き込めるようにして
    - 評価した内容はリロードしても保存してほしいので、ローカルのJSONとして管理してください。
    - GitHub Pagesアクセス時にJSONの更新は難しい場合は、読み取り専用にしてクリックしたらエラーにしてください
